# M6 Flutter 欧洲版设计

## 目标

建立独立 Flutter APP，默认 `EU` 区域，复用现有 C 端 API，先交付浏览、搜索、登录、点评、团购、预订、用户中心和通知基础链路，并为多语言、多币种、地图、支付、推送和 GDPR 提供真实可配置边界。

## 架构

- `app/lib/core`：环境配置、HTTP、会话、区域、语言、错误处理。
- `app/lib/features`：按浏览、认证、交易、预订、用户、通知拆分；页面不直接拼 HTTP。
- 默认语言根据系统选择，首批提供简体中文、繁体中文、英语；默认区域 `EU`，请求统一带 `X-Region`。
- 货币按 API 返回值格式化，不在端内猜汇率；汇率换算需要后端或第三方报价后再启用。
- Google Maps、Stripe/PayPal、FCM/APNs 通过编译/运行环境配置开启；未配置时展示明确不可用状态。

## 安全与合规

- access/refresh token 使用安全存储；Web 调试目标仅用于开发，不作为移动端生产存储口径。
- 地图定位需显式授权，拒绝后仍可按城市浏览。
- 支付结果只信后端订单状态和验签回调，客户端返回值不直接改订单。
- 隐私中心复用现有导出/删除 API，设备注册与协议留痕在后端新增表后接入。

## 验证

- Flutter 单元测试覆盖区域头、语言回退、会话与 API envelope。
- `flutter analyze` 和 `flutter test` 必须通过。
- 后续真实支付/地图/推送需要 sandbox 凭证与真机验收，仓库只提供可执行配置和失败保护。
