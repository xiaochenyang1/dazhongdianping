# M7 帖子内容闭环设计

## 1. 目标

在不混入关注关系、私信和圈子的前提下，完成社区内容的第一条真实纵向闭环：用户在 Flutter 发布图文帖子，帖子经过审核后进入公开信息流，游客可在 Flutter 与 PC Web 浏览，登录用户可点赞、收藏、评论和举报，用户可管理自己的帖子，运营人员可审核帖子，隐私导出可真实包含帖子数据。

## 2. 本阶段范围

### 包含

- 帖子：发布、本人详情、编辑、删除、公开列表、公开详情。
- 内容字段：标题、正文、内容类型、图片、话题、关联门店、关联团购、所属区域。
- 互动：点赞/取消点赞、帖子收藏、评论列表/发布、帖子举报。
- 审核：新建或编辑后进入待审核；审核通过后公开；驳回后本人可查看原因并修改重提；删除后不再公开。
- 终端：Flutter 完整互动，PC Web 只读列表/详情，管理端帖子审核。
- 治理：帖子纳入 GDPR 数据导出；删除账号到期时匿名化帖子作者信息并隐藏未公开内容。

### 不包含

- 关注/粉丝与关注流。
- 私信、拉黑、私信举报。
- 圈子及圈子成员关系。
- 转发、@提醒、评论盖楼、达人认证。
- Elasticsearch 帖子索引和真实推送；第一阶段使用 MySQL 信息流与现有通知中心能力。

## 3. 架构与边界

### 后端

新增独立 `community` 模块，控制器、服务、Mapper、请求/响应模型不与点评模块互相引用。帖子审核复用 `audit_task`，使用既有目标口径 `biz_type=4`，审核任务只保存业务 ID，帖子状态仍以 `post.audit_status` 与 `post.status` 为准。

新增表：

- `post`：帖子主体、作者快照、区域、审核与公开状态、计数。
- `post_image`：图片 URL 与顺序。
- `topic`、`post_topic`：话题及关联。
- `post_like`：用户点赞唯一关系。
- `post_comment`：一级评论；本阶段不做盖楼。
- `post_report`：帖子举报及处理状态。

帖子收藏复用 `user_favorite(target_type=2)`，同时补齐收藏 Mapper 对帖子目标的查询与列表映射，不创建 `post_favorite`。

### API

- `GET /api/c/v1/posts`：公开推荐流，按 `created_at DESC, id DESC` 游标分页，只返回审核通过且正常的帖子。
- `GET /api/c/v1/posts/{id}`：公开详情。
- `POST /api/c/v1/posts`、`PUT /api/c/v1/posts/{id}`、`DELETE /api/c/v1/posts/{id}`：Flutter 写接口。
- `GET /api/c/v1/user/posts`、`GET /api/c/v1/user/posts/{id}`：本人列表与详情，包含审核状态和驳回原因。
- `POST /api/c/v1/posts/{id}/like`：幂等切换点赞。
- `GET/POST /api/c/v1/posts/{id}/comments`：一级评论列表与发布。
- `POST /api/c/v1/posts/{id}/report`：举报。
- `GET /api/c/v1/topics`：公开话题列表。

写接口继续使用现有鉴权、`X-Region`、`Accept-Language` 和 `Idempotency-Key` 机制。

### Flutter

新增 `community` feature：

- `CommunityRepository` 负责所有帖子和互动 API。
- `CommunityFeedScreen` 提供公开信息流与刷新。
- `PostEditorScreen` 复用现有文件上传能力，支持最多 9 张图、话题和关联门店/团购 ID。
- `PostDetailScreen` 展示正文、图片、话题、点赞、收藏、评论和举报入口。
- 用户中心新增“我的帖子”，可打开编辑页查看审核结果。

首页底部导航的“Explore”调整为真实社区入口；未登录用户可浏览，写帖和互动按现有认证流程拦截。

### PC Web

新增只读 `/community` 与 `/community/posts/:id`，不暴露发布、点赞、关注或私信按钮。详情页提供下载 APP 的明确引导。

### 管理端

在现有审核中心新增帖子审核列表与详情，支持通过、驳回和填写审核备注。审核通过后帖子进入公开流；驳回后仅作者本人可见。

## 4. 数据流

1. Flutter 先通过现有 `/files/upload` 上传图片。
2. 发布帖子时保存主体、图片和话题关联，并创建 `biz_type=4` 审核任务。
3. 本人接口立即返回待审核帖子；公开接口不可见。
4. 管理端审核通过后更新帖子审核状态，公开流可见。
5. 点赞、收藏、评论和举报只允许作用于公开正常帖子。
6. 帖子编辑后清空旧驳回原因并重新进入审核；公开端在重新审核期间不可见。

## 5. 错误处理与安全

- 跨区域帖子按不存在处理，防止探测其他区域数据。
- 编辑、删除和本人详情必须校验作者。
- 图片最多 9 张、话题最多 5 个、标题最多 80 字、正文最多 5000 字。
- 关联门店/团购必须存在于当前区域；不存在时拒绝保存。
- 重复点赞、收藏和举报由唯一索引及幂等键共同保护。
- 被删除、未审核或驳回帖子不能被公开读取或互动。
- 评论和举报正文做长度校验；私信不在本阶段，因此不伪造消息导出。
