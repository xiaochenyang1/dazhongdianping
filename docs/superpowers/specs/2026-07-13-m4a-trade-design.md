# M4a 团购交易设计

## 核心状态

- 订单 `pay_status`: `0待支付 1已支付 2已退款 3部分退款`；`status`: `1正常 2已关闭`。
- 支付流水 `status`: `0处理中 1成功 2失败`；第三方交易号唯一，回调重复投递幂等。
- 券码 `status`: `1待使用 2已使用 3已过期 4已退款`；购买数量 N 生成 N 张券。
- 退款 `status`: `0申请中 1成功 2驳回`。

## 一致性

- 创建订单时原子扣减有限库存，取消未支付订单恢复库存。
- 支付回调验签后，在同一事务更新支付流水和订单，并生成券码；任何一步失败全部回滚。
- 已成功回调再次到达只返回既有结果，不重复生成券。
- 退款前锁定订单与券，存在已核销券时拒绝全单退款；模拟渠道成功后同步更新订单、退款和券状态。

## 支付适配

- 国内默认 `alipay_mock`，欧洲默认 `stripe_mock`，通过统一支付网关接口隔离渠道。
- 本地模拟渠道仍执行 SHA-256 回调签名校验，密钥来自 `APP_PAYMENT_NOTIFY_SECRET`。
- 后续替换真实 SDK 时保持订单服务和状态机不变。

## 接口

- 公开：门店团购列表、团购详情。
- 登录：创建/查询/取消订单、发起支付、退款、我的券。
- 回调：`POST /api/c/v1/pay/notify/{channel}`，不依赖用户登录但必须验签。
