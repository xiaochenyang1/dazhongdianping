# M7 官方圈子闭环设计

## 目标

实现由管理端维护的区域化官方圈子。用户可在 Flutter 浏览、加入和退出圈子，加入后可在圈子内发帖；PC Web 提供只读圈子及圈子帖子展示。圈子帖子复用现有帖子审核、互动、举报和隐私治理链路。

## 产品边界

- 圈子由管理端创建、编辑、启停和排序，普通用户不能创建或管理圈子。
- 圈子绑定 `CN` 或 `EU`，不提供跨区域或 GLOBAL 圈子。
- 游客可浏览当前区域启用的圈子、详情和公开帖子。
- 登录用户可加入或退出圈子；重复操作幂等。
- 用户必须先加入圈子才能在圈子内发帖。
- 退出圈子不删除或隐藏其历史帖子。
- PC Web 只读，不提供加入、退出或发帖操作。
- 一期不做私密圈子、入圈审批、圈主、管理员、成员禁言、邀请、群聊、付费圈子或用户创建圈子。

## 架构选择

复用现有 `post`，新增可空 `circle_id`，不创建独立圈子帖子表。

这样圈子帖子可直接复用现有发布、编辑、删除、人工审核、公开列表/详情、点赞、评论、举报、收藏、隐私导出和注销匿名化能力。圈子模块只负责圈子资料、成员关系、区域过滤和发帖资格校验，避免复制整套内容系统。

## 数据模型

### `circle`

- `id`
- `region`：`CN` 或 `EU`
- `name`：区域内唯一，长度 2-64
- `description`：长度 0-500
- `cover_url`
- `member_count`：冗余成员数
- `post_count`：冗余已发布且未删除的帖子数
- `sort`
- `status`：`1` 启用、`2` 停用
- `created_by`
- `created_at`
- `updated_at`
- `is_deleted`

约束：`UNIQUE(region, name)`；区域、状态与排序建立查询索引。

### `circle_member`

- `id`
- `circle_id`
- `user_id`
- `created_at`

约束：`UNIQUE(circle_id, user_id)`；用户维度建立索引。

### `post`

新增可空 `circle_id`。普通帖子保持为空，圈子帖子填写目标圈子 ID。

## API

### C 端

- `GET /api/c/v1/groups`：当前区域圈子分页列表，可选 `joined=true` 仅查看本人已加入圈子。
- `GET /api/c/v1/groups/{id}`：圈子详情。
- `GET /api/c/v1/groups/{id}/posts`：圈子公开帖子分页。
- `GET /api/c/v1/groups/{id}/members`：成员分页，只返回公开用户摘要。
- `PUT /api/c/v1/groups/{id}/membership`：加入圈子，幂等。
- `DELETE /api/c/v1/groups/{id}/membership`：退出圈子，幂等。

圈子详情和列表返回：`id/region/name/description/coverUrl/memberCount/postCount/joinedByCurrentUser/status`。

现有 `POST /api/c/v1/posts` 增加可空 `circleId`。传入时后端校验：

1. 圈子存在、启用且未删除。
2. 圈子区域等于当前 `X-Region`。
3. 当前用户已加入圈子。

帖子响应增加可空圈子摘要 `circleId/circleName`。

### 管理端

- `GET /api/admin/v1/circles`
- `POST /api/admin/v1/circles`
- `PUT /api/admin/v1/circles/{id}`
- `PUT /api/admin/v1/circles/{id}/status`

管理端列表遵循管理员区域权限。停用圈子后不允许新加入或发帖，但历史已审核帖子仍可公开读取；删除采用软删除。

## 计数与一致性

- 首次加入成功后 `member_count + 1`，重复加入不增加。
- 实际退出成功后 `member_count - 1`，最低为 0；重复退出不减少。
- 圈子帖子通过审核且公开时计入 `post_count`。
- 已计数帖子被删除、下架或审核状态从通过变为不通过时扣减。
- 计数更新和关系/帖子状态变更处于同一事务。

## 区域与权限

- 所有公开圈子查询强制按当前 `X-Region` 过滤。
- 不允许通过 ID 读取另一地区圈子，统一返回 404，避免泄漏跨区资料。
- 游客读取时 `joinedByCurrentUser=false`。
- 加入、退出和圈子发帖必须有用户登录态。
- 成员列表只展示状态正常、未注销用户。

## Flutter

- 社区页新增“圈子”入口，不破坏现有推荐/关注双流。
- 圈子广场展示当前区域官方圈子，突出名称、简介、成员数和帖子数。
- 圈子详情展示封面、简介、加入/退出按钮、成员入口和帖子列表。
- 未登录用户点击加入或发帖时显示登录引导，不调用受保护接口。
- 已加入用户可从圈子详情打开现有帖子编辑器，自动携带 `circleId`。
- 退出后历史帖子继续显示，但发帖入口切换为加入提示。
- “我的”页面增加“我的圈子”入口。

视觉方向沿用现有暖色本地生活风格，使用有辨识度的封面卡片、清晰成员/帖子数字和不抢内容的加入按钮；不新增图片生成资产，缺省封面使用代码内渐变与首字图形。

## PC Web

- 新增只读圈子列表和详情路由。
- 圈子详情展示简介、成员数、帖子数和公开帖子。
- 不展示加入、退出或发帖按钮。
- 圈子帖子仍链接到现有帖子详情和公开用户主页。

## 管理端

- 新增圈子管理列表和编辑表单。
- 支持按区域、状态搜索，创建、编辑、启停和排序。
- 名称冲突、跨区操作和非法状态返回真实错误，不做静默覆盖。
- 不在一期提供成员逐个管理或内容审核入口；帖子继续进入现有审核任务。

## 隐私与注销

- `follows` 与 `messages` 之外新增 `circles` 隐私导出模块，包含本人加入的圈子与加入时间。
- 账号注销时删除全部 `circle_member` 关系并同步扣减成员数。
- 历史圈子帖子继续按现有帖子注销治理匿名化，不因退出圈子或注销删除公共内容。

## 错误处理

- 圈子不存在、停用、已删除或跨区域：404。
- 未加入圈子发帖：409，提示先加入圈子。
- 非法名称、简介、状态或分页参数：400。
- 重复加入和重复退出返回 200，并返回当前真实成员状态与成员数。
- 管理端区域越权沿用现有管理员权限处理。

## 测试范围

### 后端

- 圈子列表、详情、成员和帖子按区域过滤。
- 游客读取与登录用户 `joinedByCurrentUser`。
- 加入/退出幂等和成员计数。
- 未加入、跨区域、停用圈子发帖失败。
- 圈子帖子复用审核、互动、举报和删除流程。
- 帖子公开状态变化正确维护 `post_count`。
- 管理端 CRUD、启停、排序、区域权限与名称唯一约束。
- `circles` 隐私导出与注销关系清理。

### Flutter

- 圈子广场、详情、加入/退出、成员列表、圈子帖子与“我的圈子”。
- 游客登录引导不误发受保护请求。
- 圈子详情发帖正确携带 `circleId`。
- 加入/退出乐观状态失败时恢复。

### PC Web

- 圈子列表和详情只读展示。
- 圈子帖子跳转正确，不出现写操作。

## 验收标准

1. 管理员能创建当前区域官方圈子并启停。
2. Flutter 用户可浏览、加入、退出，并在加入后发圈子帖。
3. 圈子帖子完整复用现有审核与互动链路。
4. CN/EU 圈子和帖子不串区。
5. PC Web 可只读查看圈子和帖子。
6. 隐私导出和账号注销正确治理成员关系。
7. 全量 CI、Flutter analyze 和构建全部通过。
