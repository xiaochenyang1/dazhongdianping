# Web 隐私中心设计

## 背景与目标

后端已经实现隐私概览、数据导出任务、ZIP 下载、账号删除申请、冷静期撤销和到期匿名化，但 `web` 没有用户入口。目标是补齐 `/user/privacy`，让已登录用户能在浏览器完成导出与注销闭环，并把真实状态、限制和风险说清楚。

## 方案选择

采用独立隐私中心页面，而不是继续膨胀 `ProfileView.vue`。资料页只提供入口；隐私页负责接口编排、任务状态、下载和危险操作。这样导出与注销可以独立测试，后续补协议留痕或二次确认时也不需要拆旧页面。

## 页面结构

1. 顶部说明区：展示“数据可带走、删除有冷静期”的核心规则。
2. 导出区：展示每日次数、文件有效期、模块选择、创建按钮和最近任务列表；仅状态为“可下载”的任务显示下载按钮。
3. 注销区：展示冷静期、当前任务状态、校验方式、账号、验证码/密码和原因；冷静期中的任务可撤销。
4. 资料页入口：在现有用户中心动作区增加“隐私中心”。

## 数据流与错误处理

- 页面进入时并行读取隐私概览与导出任务列表。
- 创建导出任务后刷新概览和列表。
- 下载使用带登录态请求获取 Blob，再由浏览器生成临时下载链接；不能直接使用普通 `<a href>`，否则缺少 `Authorization`。
- 删除申请支持密码或验证码校验；验证码复用现有 `sendAuthCode(scene='delete')`。
- 撤销成功后刷新概览。
- 所有接口错误显示后端消息与 `traceId`；按钮在请求期间禁用，避免重复提交。

## 视觉与可用性

沿用现有暖色玻璃质感和品牌色。导出区偏中性，注销区使用低饱和危险色，不靠夸张弹窗吓人。标题使用平衡换行，说明文字使用自然换行；任务编号、限额和状态数字使用 tabular numbers；所有按钮至少 40px 高，并提供 `active: scale(0.96)` 的触感反馈。

## 测试与验收

- Vitest 验证隐私服务调用路径、页面加载、创建导出、下载、发送注销验证码、提交删除和撤销。
- 路由测试验证游客访问 `/user/privacy` 会触发登录拦截。
- Playwright 真实后端 E2E 验证：登录 → 创建导出 → 下载可用 → 创建删除申请 → 撤销成功。
- 完整运行 `verify-all.ps1`、浏览器冒烟和真实后端 E2E。
