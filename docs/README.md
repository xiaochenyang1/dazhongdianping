# 大众点评(仿)项目文档总览

> 文档状态:主文档、实施计划、测试、上线、采购、值班、自动化与执行台账模板已补齐,当前已可支撑本地继续开发
> 最后更新:2026-07-21

## 1. 项目概述

这是一个仿大众点评的本地生活平台方案文档集,当前覆盖四类端:

- C 端 PC Web:国内用户浏览、搜索、点评、交易、榜单。
- C 端 APP:海外华人为主,重点做欧洲区本地生活与社区社交。
- B 端商户工作台:商户入驻、门店管理、团购、核销、经营数据。
- 管理端:审核、运营配置、基础数据、RBAC、审计。

项目采用一套后端 + 一套数据库 + `region` 隔离业务数据的方式承载 `CN` 和 `EU` 两个区域。

## 4. 推荐阅读顺序

1. 先看 `需求文档.md`,确认项目到底要做哪些能力,别上来就想写表。
2. 如果你现在最关心的是“前后端到底联到哪了”,先看 `前后端联调记录.md`,别靠聊天记录拼结论。
3. 再看 `当前已完成功能与SQL导入说明.md`,先搞清楚当前到底已经做完了啥、标在哪些文档里、对应哪套 SQL、怎么导库、用哪个账号演示。
4. 然后看 `系统架构设计.md` 和 `业务流程与状态机.md`,把跨模块关系和关键状态搞清楚。
5. 接着看 `权限矩阵.md`、`接口设计.md` 和 `数据库设计.md`,把权限、接口和存储落到可执行层。
6. 开工前补看 `M1-M2实施计划与验收清单.md`、`第三方供应商选型表.md`、`预算与采购开通清单.md`、`自动化测试与发布流水线说明.md`,别排期、采购、CI/CD 全靠感觉。
7. 联调和上线前看 `测试清单与验收用例.md`、`上线准备清单.md`、`值班与故障响应手册.md`、`执行版联系人与台账模板.md`,别线上再临时对表。
8. 凡是术语不统一、状态定义打架、还没拍板的地方,统一回 `术语与枚举规范.md` 与 `文档差异与待确认清单.md`。
