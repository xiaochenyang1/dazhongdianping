## 0. 区域架构(全局原则,最重要)

> 一套代码、一个后端、一份数据库,通过「区域(region)」隔离国内与海外数据。这是全站的地基,所有表和接口都要遵守。

- **区域枚举**:`CN`(国内,PC 端默认)/ `EU`(欧洲,APP 端默认),预留后续扩展(如 `US`、`AU`)
- **请求区域(`request_region`)**:由端侧请求头 `X-Region` 表达。当前 `Web` 默认 `CN` 但已支持切换 `CN / EU`,`APP` 规划默认 `EU`;单次请求始终只在一个区域上下文内执行
- **数据区域(`data_region`)**:商户、城市/商圈、点评、团购、榜单、运营位等业务数据都带 `region` 字段;新增区域只加数据,不改代码
- **用户偏好区域(`preferred_region`)**:仅表示账号默认浏览偏好或后续营销画像,**不作为访问控制字段**
- **按 region 切换的能力**:
  - 币种:CN → CNY;EU → EUR / GBP
  - 地图:CN → 无地图;EU → Google Maps
  - 支付:CN → 支付宝/微信;EU → Stripe / PayPal / SEPA(+ 支付宝/微信兜底)
  - 语言:CN → 简体中文;EU → 中(简/繁)+ 英
  - 合规:EU → GDPR / SCA / VAT
- **共用不分区**:用户账号体系、鉴权、商户端、运营后台是**跨区域统一**的(一个用户可同时在 CN/EU 有行为;运营后台按区域权限切区管理)
- **商户经营区域**:一期按**单商户主体单区域经营**设计,一个商户主体只能挂在一个 `region`;跨区经营留后期扩展
- **搜索**:ES 索引按 region 过滤(可用 region 做 routing / 独立索引,后续定)

## 1. 项目概述

一个仿"大众点评"的本地生活服务平台,包含商户信息展示、用户点评、团购优惠、预订等核心能力。

分两个端交付:

| 端 | 技术栈 | 面向人群 | 说明 |
|---|---|---|---|
| PC 端(Web) | Vue3 + TypeScript + Vite | 国内用户 | 优先交付,先做 |
| APP 端 | Flutter | **海外华人** | 后续交付,基于同一套后端 |
| 后端 | Java + SpringBoot + MyBatis | — | 两端共用一套 API |

技术组件:
- 数据库 **MySQL**(主存储)+ **Redis**(缓存、会话、验证码、限流、热点榜单)
- 搜索 **Elasticsearch**(数据量大,商户/点评/团购全文检索、聚合筛选、搜索联想)
- 对象存储(图片):国内 OSS / 海外 S3 + CDN(暂不含视频)
- 地图:**仅 APP 端(欧洲)用 Google Maps**;PC 端不使用地图(地址/商圈用文字 + 分类筛选)
- 实时通信:**WebSocket**(PC 端站内实时通知、私信);APP 端叠加 **FCM / APNs** 推送
- 消息队列(可选,后期):订单超时关闭、异步核销、内容审核、推送 —— RabbitMQ / RocketMQ

## 2. 业务领域模型(核心概念)

先对齐几个贯穿全站的核心对象,后面所有功能都围绕它们展开:

- **用户(User)**:注册/登录、个人主页、等级、收藏、订单。
- **商户/店铺(Shop/POI)**:餐厅、娱乐、丽人、酒店等实体,含地理位置、营业时间、图片、菜单。
- **分类(Category)**:美食、休闲娱乐、丽人、亲子、结婚、酒店、电影演出等多级分类。
- **地理(City / 商圈 / 定位)**:城市 → 商圈/行政区 → LBS 附近。
- **点评(Review)**:评分(口味/环境/服务)+ 文字 + 图片/视频。
- **团购/优惠(Deal/Coupon)**:套餐、代金券、优惠券。
- **订单(Order)**:团购下单、预订、支付。
- **榜单(Rank)**:必吃榜、好评榜、热门榜。

## 4. 后端(SpringBoot + MyBatis)模块划分

按功能域拆分模块,先粗分,后面细化到接口:

| 模块 | 职责 |
|---|---|
| 用户中心 | 注册登录、鉴权(JWT)、个人信息、等级积分、收藏、设备管理 |
| 商户服务 | 商户 CRUD、分类、地理/商圈、营业信息 |
| 搜索服务 | 关键词搜索、筛选排序(ES 或 MySQL) |
| 点评服务 | 点评发布、评分聚合、点赞评论、内容审核 |
| 交易服务 | 团购、优惠券、订单、支付对接、券码核销、**预订/订座** |
| 内容运营 | 首页推荐、Banner、榜单、运营位配置、运营活动/专题 |
| 消息通知 | 站内信、**WebSocket 实时推送(PC)**、FCM/APNs(APP) |
| 文件服务 | 图片/视频上传、对象存储 |
| 数据接入 | **种子商户批量导入**(运营录入 / Excel / 模板)、MySQL→ES 同步 |
| 商户服务(B端) | 商户入驻、员工角色、资质审核流、门店/团购/预订管理、商家回复、经营看板 |
| 运营后台 | 审核中心、基础数据、用户/订单管理、运营配置、RBAC、审计日志 |

横切能力:统一鉴权、统一异常、参数校验、日志、幂等、限流、接口文档(Swagger/Knife4j)、隐私任务留痕。

> 说明:后端一套服务,对外提供三类 API —— C 端(PC/APP 用户)、B 端(商户端)、管理端(运营后台),按角色鉴权隔离。
