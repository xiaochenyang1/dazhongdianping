## 1. 文档目标

这份文档只干三件事:

- 把一期要花钱的地方拆明白。
- 把供应商账号、域名、证书、模板、回调这些开通前置项列清楚。
- 把谁去买、谁去配、谁续费、谁背锅这事说清楚。

别指望靠脑子记住这些采购细节。真到要上线的时候,漏一个 sender、少一个 webhook、忘一个证书续期,项目就能原地给你上演行为艺术。

## 2. 预算原则

- **先按 M1/M2 必需项采购**,别一上来把后面半年都买满。
- **生产和测试分开估算**,避免测试乱烧生产资源。
- **按区域拆预算**:`CN`、`EU` 不混着记。
- **按能力建预算项**,别只记“云成本”这种没法追的烂账。
- **所有金额以供应商控制台/合同/采购单为准**,本文只负责预算结构,不固化价格。

## 8. 供应商专项注意事项

### 8.1 邮件

- 发信域名要单独管理,别和主业务域名乱搅。
- `SPF/DKIM/DMARC` 都得补齐。
- 测试环境与生产环境发信地址建议分开。
- 发信额度和退信/投诉率要有人盯。
- Amazon SES 要注意:
  - 先验证发信地址或域名 Identity。
  - 生产发送前要确认是否仍在 sandbox。
  - 发送额度按 Region 单独看。
- 阿里云 DirectMail 要注意:
  - 发信域名、发信地址、SMTP 密码都要先准备。
  - 不同区域的发信域名和地址互不通用。

### 8.2 短信

- 模板和签名提前准备,别等联调当天才提审。
- 国际短信要核对国家覆盖、发送用途和 sender 类型。
- 预算要按峰值预留,验证码类最容易突然烧起来。
- Twilio 要注意:
  - 先选 sender 类型,再走对应的注册或验证流程。
  - 状态回执回调 URL 要提前定好,别上线时现补。

### 8.3 对象存储与 CDN

- 公有读图片和私有导出包分 Bucket 或至少分前缀策略。
- CDN 回源白名单、缓存 TTL、刷新权限都要定清楚。
- 导出包要单独生命周期策略,别和图片一锅煮。
- CloudFront / 阿里云 CDN 都要提前确认:
  - 加速域名归属验证
  - HTTPS 证书
  - 回源地址
  - 缓存策略
  - 日志投递

### 8.4 审核与推送

- 审核额度和 QPS 要提前确认。
- FCM/APNs 证书、Key、Bundle ID、包名别到上线前一天才对。
- 审核回调、推送失败回执都要有人接日志和告警。
- FCM 要注意:
  - Firebase 项目和服务账号要先建好。
  - 服务端发送逻辑建议统一走 Admin SDK 或 HTTP v1。
- APNs 要注意:
  - 明确走 token-based 还是 certificate-based。
  - Provider Server 需要维护 HTTP/2 + TLS 连接,并管理 device token。

## 9. 续费与到期提醒

建议给这些东西都建提醒:

- [ ] 域名到期
- [ ] HTTPS 证书到期
- [ ] Apple 推送证书/Key 到期
- [ ] 云资源包/套餐到期
- [ ] 短信包/邮件配额不足
- [ ] 支付证书或平台证书到期

建议提醒节奏:

- 提前 `30` 天
- 提前 `15` 天
- 提前 `7` 天
- 到期前 `1` 天
